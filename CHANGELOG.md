# Changelog

本项目所有显著变更记录于此文件。

格式遵循 [Keep a Changelog 1.1.0](https://keepachangelog.com/zh-CN/1.1.0/)，
版本号遵循 [Semantic Versioning 2.0.0](https://semver.org/lang/zh-CN/)。
三端（后端 · 游戏端 · 前端）各自独立维护版本号。

> **阅读约定**：`[Unreleased]` 段面向开发者，可含实现细节与提交引用；
> 已发版段落（`vX.Y.Z`）面向**使用者**，只讲「能感知到什么变化」，用自然语言、尽量精简。

## [frontend-v0.6.1] - 2026-07-11

### Fixed

- **打开大项目（上百行）更快**：首屏渲染从约 2.6s 降到约 1.2s；二次进入同一项目命中本地缓存几乎瞬开，不再每次等加载。
- 拖拽调整过的列宽在刷新或自动刷新后不再被重置。

---

## [htcmc_auth-v0.6.0] - 2026-07-11

### Added

- **一键提交快捷命令**：新增 `!!submit`，直接重开上次查看的项目并扫描背包一次性上交；`!!submit <编号>` 可指定项目。与 `!!PCH sheet submit` 等效。

### Fixed

- **一键提交回执精简**：提交后只逐行显示与你相关的跳过（你认领的、已备齐 / 无需求的），其余跳过项折叠为末尾一行计数，不再被无关行刷屏淹没。

---

## [backend-v0.6.0] - 2026-07-10

### Added

- **子物品嵌套行**：材料支持挂子件（如「机械臂」配「齿轮×2」），系统按单位用量自动算出子件需求；认领或交付父行时自动级联其所有子行。单位用量支持小数（如 0.5）。
- **材料行智能排序**：打开项目时，你认领的、你参与的行排前面，最缺的数量优先；CSV 导出仍是自然序。
- **材料行搜索**：项目详情可按物品名 / 注册名过滤材料行。

### Fixed

- 修改材料名不再误建重复行；新建同名材料会被拦截并提示。
- 调整子件用量导致需求超过已交量时，行状态不再卡在「已完成」，会正确回退为「进行中」。
- 删除带子件的材料行时，认领人 / 贡献者会收到子件被删的通知。

---

## [frontend-v0.6.0] - 2026-07-10

### Added

- **子物品树状展示**：材料行按父子关系分组显示，子件缩进排列，可在行内直接增删与编辑子件。

### Fixed

- 编辑材料保存时改名生效、不再重复建行。
- 打开上百行的大项目时不再每秒卡顿，自动刷新不会丢失正在编辑的草稿。
- 添加子物品成功后弹窗正常关闭。

---

## [htcmc_auth-v0.5.0] - 2026-07-10

### Added

- **子物品命令**：新增 `!!PCH sheet addsub` / `delsub` / `setsub` 管理子物品；查看项目时子件缩进显示，父行标注子件数量。
- **项目分页查看**：`!!PCH sheet view` 每页 30 行，避免超出聊天框截断；支持 `-p <页码>` 翻页。
- **项目关键词搜索**：`!!PCH sheet view <id> -s <关键词>` 按物品名 / 注册名过滤，底栏常驻「搜索」按钮。

### Changed

- 材料数量显示改为友好单位（个 / 组 / 盒），如 1728 个显示为「1 盒」，与 Web 端一致。
- 行显示精简：进度行显示「当前/需求」，锁定行只显示需求值。

### Fixed

- 修复 `addsub` / `setsub` 指定 progress 模式时实际仍建成 lock 行的问题（v0.3.0 起遗留）。

## [htcmc_auth-v0.4.0] - 2026-07-06

### Added

- **一键提交**：查看项目时点按钮即可扫描整个背包，把匹配的材料一次性上交（支持潜影盒内的物品）。
- **快速重开上次项目**：直接输入 `!!sheet` 重开上次查看的项目，或 `!!sheet <编号>` 打开指定项目；`!!PCH sheet last` 同效。
- **更聪明的项目列表**：默认只显示进行中的项目；自己参与的（作为创建者 / 认领人 / 贡献者）排在最前；每行标注所处阶段（收集中 / 建造中）。
- **列表筛选简写旗标**：`!!sheet list` 支持 `-m`(我的) / `-c`(收集中) / `-t`(建造中) / `-a`(已归档) / `-l`(全部)，可组合如 `-ma`。
- **拥有者改物品 ID**：项目拥有者可在行上点「改ID」直接修改材料对应的游戏物品（缺省读取手持物品）。
- **查看界面美化**：物品列表新增分节标题，按钮右对齐、底栏按钮居中，显示更整齐。

### Fixed

- 空项目（无任何物品行）不再错误显示「一键提交」按钮，并补回物品列表的分节标题。

## [backend-v0.3.0] - 2026-07-03

- **投影文件一键生成材料清单**：上传 `.litematic` 投影文件，系统自动解析其中的方块并翻译成中文名，一键生成项目材料清单（支持原版与 Create 模组物品）。

## 版本化策略

三端各自独立 SemVer，按 [`CONTRIBUTING.md`](./CONTRIBUTING.md) §4 维护。

| 组件 | tag 约定 |
|---|---|
| 后端 | `backend-vX.Y.Z` |
| 游戏端（MCDR 插件） | `htcmc_auth-vX.Y.Z`（v0.4.0 起改名，符合 MCDR PluginCatalogue 合法格式；历史 `mcdr-v0.3.0` 前向兼容保留） |
| 前端 | `frontend-vX.Y.Z` |

发版流程：

1. 把 `[Unreleased]` 段浓缩为面向使用者的自然语言（去掉实现细节与提交哈希）。
2. 固化为 `## [<组件>-vX.Y.Z] - YYYY-MM-DD` 段，置顶于已发版段落上方。
3. 重置 `[Unreleased]` 段为空占位。
4. 打对应 tag。

> MCDR tag 改名背景见 [`Docs/Reports/mcdr-publishing-strategy.md`](./Docs/Reports/mcdr-publishing-strategy.md)。
